## **1. Grundprinzipien: Open-Source Lizenzen und Daten**

Wenn du ein Portal starten willst, das Inhalte oder Daten anderer Open-Source-Portale nutzt, musst du unterscheiden:

### **A) Software-Lizenzen**

* **GPL / AGPL / LGPL**

  * Freie Nutzung und Weitergabe, aber bei Modifikationen musst du **den Quellcode offenlegen**.
  * AGPL verlangt das auch, wenn du über das Web darauf zugreifst.
* **MIT / BSD / Apache 2.0**

  * Sehr permissiv: Du kannst die Software auch in proprietäre Produkte integrieren, meistens mit **Lizenz- und Copyright-Hinweis**.
* **Eclipse Public License, MPL**

  * File-basiert: Geänderte Dateien müssen offen sein, anderes darf proprietär bleiben.

### **B) Inhalte / Daten**

* **CC0 / Public Domain** – frei nutzbar ohne Einschränkung.
* **Creative Commons Lizenzen (BY, BY-SA, BY-NC)**

  * BY: Namensnennung erforderlich.
  * BY-SA: Weitergabe unter derselben Lizenz erforderlich.
  * BY-NC: Nicht-kommerzielle Nutzung.

> Für dein Portal musst du **jede Quelle prüfen**, ob die **Software oder Daten / Inhalte** unter einer CC-Lizenz, GPL/MIT/Apache etc. stehen.

## **2. Ansatz: Community License für dein Portal**

Wenn du eine **Community License für den Aufbau eines Portals** erstellen willst:

1. **Fokus auf Aggregation & Verlinkung**

   * Viele Open-Source-Portale (z. B. F-Droid, GitHub) erlauben das **Verlinken**, solange du **ihren Namen / Logo korrekt angibst**.
   * Du musst **nicht unbedingt deren Software kopieren**, sondern verlinkst nur auf Daten, Repos, Packages.

2. **Datenbereitstellung & Aggregation**

   * Wenn du z. B. die **App-Liste von F-Droid** aggregierst, musst du:

     * **F-Droid Lizenz prüfen** (F-Droid selbst ist unter GPLv3, Metadaten unter CC-BY-SA)
     * **Quellenangabe / Attribution** bereitstellen

3. **Eigene Community-Lizenz formulieren**

   * Basis: Permissive Lizenz (ähnlich Apache 2.0 oder MIT)
   * Kernpunkte:

     * Alle Beiträge / Aggregationen bleiben **frei nutzbar für Community-Mitglieder**
     * **Attribution** zu Original-Quellen erforderlich
     * **Modifikationen / Weitergabe** erlauben, aber **unter derselben Community-Lizenz**, falls gewünscht
     * Optionale **“Non-Commercial / Commercial Use”** Regeln

## **3. Praktisches Vorgehen: Schritt für Schritt**

1. **Inventar erstellen**

   * Jede Quelle prüfen: Software vs. Daten / Content
   * Lizenz notieren (GPL, MIT, CC-BY-SA, Public Domain)
   * Besonderheiten für Web/API-Zugriff dokumentieren

2. **API oder Metadaten sammeln**

   * Wo möglich: **öffentliche APIs nutzen**, anstatt Daten lokal zu kopieren
   * Beispiele: F-Droid Repo JSON, GitHub REST API, VS Code Marketplace API

3. **Rechtssichere Attribution**

   * Für jede Quelle ein **Namensnennungssystem** im Portal
   * Lizenz-Informationen sichtbar machen

4. **Community-Lizenz definieren**

   * Startpunkt: **Apache 2.0 oder MIT**, erweitert mit Attribution-Pflicht
   * Option: “Community License for Aggregation”

     * Inhalte & Software der Community-Mitglieder frei verfügbar
     * Weitergabe und Anpassung erlaubt, mit Attribution
     * Beiträge bleiben **unter der Community License**, wenn man will

5. **Technische Umsetzung**

   * Portal als **Aggregator / Verzeichnis** aufbauen
   * Inhalte über APIs / Scraper legal abrufen
   * Lizenzhinweise für jedes Item anzeigen

## **4. Beispielhafte Formulierung einer Community License**

**Community License – Prinzipien:**

* Attribution: Jede verlinkte Quelle wird genannt
* Share-Alike: Modifikationen am Portal (nicht die Originalsoftware) müssen unter derselben Lizenz bleiben
* Nutzung: Alle Mitglieder können Inhalte & Links frei nutzen
* Klarstellung: Keine proprietäre Sperre für Inhalte der Community

> Diese Lizenz ist **keine Standard-OSS-Lizenz**, sondern **eine Vereinbarung für Community-Zusammenarbeit**, die mit bestehenden Open-Source-Lizenzen kompatibel sein sollte.

💡 **Wichtiger Hinweis:**

* **Juristische Prüfung:** Jede Nutzung von Software oder Daten erfordert eine **Lizenzprüfung** – besonders bei GPL/AGPL.
* **Attribution & Kompatibilität** sind die einfachsten Wege, dein Portal legal aufzubauen.
* APIs und Metadaten aggregieren ist **meist sicherer als Kopieren**, solange du die Lizenzbedingungen der Originalquelle beachtest.
